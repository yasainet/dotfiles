# RULES.md ドキュメントスタイルガイド

`~/Projects/eslint/docs/RULES.md` は全 ESLint ルールの唯一の正式ドキュメントである。ルールセクションの追加・編集時は以下の規約に厳密に従うこと。

## セクション構成

各ルールセクションには、以下のフィールドをこの順序で必ず含めること:

```markdown
#### ルール名 (`eslint-rule/name`)

- **説明**: ルールの内容と存在理由
- **対象**: 対象ファイルの Glob パターン

- **OK 例**:
  \`\`\`ts
  // file-path/example.ts
  正しいコード
  \`\`\`

- **NG 例**:

\`\`\`ts
// file-path/example.ts
違反コード // 違反理由のインラインコメント
\`\`\`
```

## フォーマットの選択

| 条件                                        | フォーマット              |
| ------------------------------------------- | ------------------------- |
| 1 行の例（1 ステートメント）                | インライン: `` `code` ``  |
| 複数行の例（2 ステートメント以上）          | コードブロック: ` ```ts ` |
| 複数ディレクトリ/パターンで同一ルールを共有 | OK/NG 列のテーブル        |

**インライン**形式の場合、3 つの例をカンマ区切りまたは箇条書きで列挙する:

```markdown
- **NG 例**: `bad1`, `bad2`, `bad3`
- **OK 例**: `good1`, `good2`, `good3`
```

**コードブロック**形式の場合、各例をファイルパスコメント付きの個別ブロックにする。

**テーブル**形式の場合:

```markdown
| ディレクトリ | パターン  | OK 例 | NG 例 |
| ------------ | --------- | ----- | ----- |
| `dir/`       | `pattern` | `ok1` | `ng1` |
|              |           | `ok2` | `ng2` |
|              |           | `ok3` | `ng3` |
```

## フィールドラベル

日本語ラベルを統一して使用する:

| フィールド | ラベル      |
| ---------- | ----------- |
| 説明       | `**説明**`  |
| 対象       | `**対象**`  |
| NG 例      | `**NG 例**` |
| OK 例      | `**OK 例**` |

## コードブロックの規約

- 1 行目に必ずファイルパスコメントを含める: `// services/server.service.ts`
- NG の理由をインラインコメントで説明する: `// lateral 違反`
- 抽象的な名前（`foo`, `bar`）ではなく、現実的な識別子（`getComics`, `serverService`）を使う
- 例は最小限にし、関連する行のみを示す

## 見出し階層

```
## 環境セクション (common / Next.js / Node.js / Deno)
### ソースファイルグループ (rules.mjs / naming.mjs / layers.mjs / imports.mjs)
#### 個別ルール
##### サブセクション（環境固有のバリエーションがある場合のみ）
```

## 記述スタイル

- 説明は日本語で記述する
- コードコメントと識別子は英語で記述する
- 本文は 1 文で簡潔に説明し、必要に応じて 2 文目で理由を補足する
- 箇条書きの末尾に句点を付けない
