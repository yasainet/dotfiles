## Phase: Implement

- 承認済みの plan を実装に移す
- コードを書く前に「何をもって動いたと判定するか」を必ず宣言する

## Steps

1. plan を読む

   - `$ARGUMENTS` で指定された `docs/plans/*.md` を読む
   - 指定がなければ最新の plan を提示して確認を取る
   - plan に疑問があれば着手前に相談する

2. 検証方法を宣言する

   - テスト可能なロジック → TDD で実装する
   - 設定・script・UI → 実行ベースの検証手順を明示する
   - どちらか迷う場合は理由とともにユーザーに確認する

3. TDD の場合は red → green → refactor を厳守する

   - 失敗するテストを先に書き、実行して失敗を確認せよ
   - テストを通す最小限の実装を書き、実行して成功を確認せよ
   - テストより先に書いた実装コードは破棄せよ

4. 機械的なタスクは subagent に委譲し、model を必ず明示する

   - 判断を伴う実装は sonnet、機械的な変更は haiku を指定する
   - plan に書かれた該当タスクの内容だけを渡し、セッション履歴は渡さない

5. 宣言した検証を全て通してから完了を報告する

   - 検証結果（コマンドと出力の要点）を報告に含める

> [!NOTE]
>
> - 検証を通さずに完了を宣言するな
> - 行き詰まったら同じ方法で再試行せず、状況を報告して相談せよ
> - plan にないスコープの変更に気づいたら、実装せず報告せよ
