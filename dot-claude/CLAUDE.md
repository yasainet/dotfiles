# Workflow

> [!IMPORTANT]
>
> - 不確かなことは、推測せずに確認と調査をせよ
> - 挨拶、前置き、は不要であり結論ファーストで回答せよ
> - 指摘すべきことは率直に指摘せよ
> - 次の手順に従い実行する: 要約、調査、計画、実行、検証

1. 要約: ユーザーのリクエストを要約し、目的に齟齬がないか確認せよ
2. 調査: ゴール達成に必要な関連ファイル、Web、ドキュメントを検索せよ
3. 計画: `EnterPlanMode` ツールを呼び出して Plan モードに入り、TODO を提案せよ
4. 実装: TODO リストに従って実装せよ
5. 検証: プロジェクトの `CLAUDE.md` で定義された検証手順を実行せよ
